## 📂 โครงสร้างภายในโปรเจกต์ (Project Structure)

โปรเจกต์นี้มีโค้ดแบ่งออกเป็น 2 รูปแบบหลัก เพื่อการเลือกใช้งานตามความเหมาะสม:

### 1. `static-version` (โฟลเดอร์สำหรับคลาวด์ Static เช่น Netlify / Vercel)
* **การทำงาน:** เป็นเว็บแบบไร้เซิร์ฟเวอร์หลังบ้าน (Serverless / Jamstack) เหมาะสำหรับการขึ้นระบบแบบฟรี 100%
* **ระบบแอดมินหลังบ้าน:** ข้อมูลการแก้ไขจะเซฟลง LocalStorage ของเครื่องเบราว์เซอร์ผู้ใช้งาน และสามารถกดดาวน์โหลดไฟล์ `database.json` ไปทับในโฟลเดอร์เพื่อ Deploy ใหม่ได้
* **ระบบจดหมายข่าว:** ดึงภาพและข้อมูลจากโฟลเดอร์สาธารณะของ Google Drive โดยอัตโนมัติ (ตรวจจับโดเมน Netlify และ Vercel เพื่อใช้ API Serverless Proxy อัตโนมัติ ปลอดภัยและแก้ปัญหา CORS)

### 2. `server-version` (โฟลเดอร์สำหรับรัน Node.js + Express Server)
* **การทำงาน:** เหมาะสำหรับการโฮสต์ผ่านเซิร์ฟเวอร์จริง หรือคอมพิวเตอร์เซิร์ฟเวอร์ภายในโรงเรียน
* **ระบบหลังบ้าน:** เขียนบันทึกข้อมูลทับไฟล์ `data/database.json` บนเซิร์ฟเวอร์ทันทีแบบ Real-time และสามารถอัปโหลดไฟล์ภาพเก็บลงเครื่องเซิร์ฟเวอร์ได้โดยตรง

## 🔑 ข้อมูลการเข้าสู่ระบบสำหรับผู้ดูแลระบบ (Admin Login)

เมื่อเปิดเข้าสู่หน้าจัดการระบบหลังบ้านที่ลิงก์ `/admin` (เช่น `http://localhost:3000/admin` หรือ `https://your-site.netlify.app/admin`) บัญชีผู้ดูแลระบบเริ่มต้นสำหรับลงชื่อเข้าใช้คือ:

* **ชื่อผู้ใช้ (Username):** `admin`
* **รหัสผ่าน (Password):** `admin`

*(แนะนำให้ทำการเปลี่ยนรหัสผ่านเพื่อความปลอดภัยหลังจากเข้าสู่ระบบครั้งแรกเรียบร้อยแล้ว ในแถบเมนู "เปลี่ยนรหัสผ่าน" ด้านซ้ายมือ)*
